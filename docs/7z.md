# Optional 7z bridge (Node-only)

Bytefold exposes a minimal opt-in bridge to the external `7z` binary. It is not used by default.

## API

```js
import { listWith7z, extractWith7z } from '@ismail-elkorchi/bytefold/node/external';

const list = await listWith7z('/path/to/archive.7z');
console.log(list.entries);

await extractWith7z('/path/to/archive.7z', '/tmp/output');
```

## Behavior

- Detects `7z` or `7za` using `command -v`.
- Throws `ExternalToolError` with code `EXTERNAL_TOOL_MISSING` when the tool is absent.
- Never logs passwords or arguments.

## Security implications

- Runs an external process; treat as a boundary crossing.
- Only use on trusted hosts with vetted toolchains.
- Prefer Bytefold-native parsing for untrusted inputs.

## Tests

`npm run test:external-7z` will:

- Skip if `7z` is missing.
- Otherwise list and extract a small ZIP using the external tool.
